# Security policy

## Supported versions

Security fixes are provided for the latest released version and the current
`master` branch. Older releases are not supported.

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability. Use
[GitHub private vulnerability reporting](https://github.com/Dejniel/TiMini-Print/security/advisories/new)
or email `timini-print-security@wtrymiga.pl`.

Include, where possible:

- the affected version and operating system;
- the affected transport, printer model, and profile;
- a minimal reproduction and expected impact;
- sanitized logs or files needed to reproduce the problem.

Remove Bluetooth addresses, serial numbers, credentials, personal data, and
other unique device identifiers before submitting a report. Do not attach
manufacturer applications, firmware, SDKs, decompiled sources, or other
proprietary material.

You should receive an acknowledgement within seven days. Details may be kept
private until a fix and coordinated disclosure are ready.
